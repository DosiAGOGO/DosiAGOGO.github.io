Springboot项目实战-社区建立

配置环境：

首先要配置jdk8，maven，下载Idea Ultimate版本，（社区版本打开springboot好像有点麻烦），一般Ultimate版本可以通过学校邮箱注册，免费使用。安装方法可见我的另一篇博客。配置好以后直接新建项目即可使用springboot，本项目是做一个基于Springboot框架的Web社区，详细教程可以参考码酱社区。本文也是基于此教程做的整理和归纳。

  • 配置jdk8，maven，Idea Ultimate。

  • 安装完成后，点开idea。

创建工程：

  • 选择new project

  • 左侧选择Spring Initializr，默认会置于 start.spring.io ,SDK选择1.8。点击next。

  • 配置Metadata

    • Group：是所要创建网页的倒叙，比如com.baidu.www。这里我输入的是cc.smcoco.community

    • artifact：和group共同组成最后一行package。artifact我输入的是community

    • name：修改成community

    • Description：描述随意。

    • 其他随意。点击next。

  • Dependencies

    • 选择 Web-Spring Web 点击next

  • name修改为community，点击next

Springboot工程文件简介（基础认知）：

图中：

  1. 根目录： 在最根部目录(community)处点击右键 -> mark as -> source root。此时会发现其子目录部分文件夹图标内多了一个圆点（package图标）。如果要在子目录中创建package，必须设定根目录，否则无选项。

  2. 子目录中src： 表示源码。src.main.java.cc.smcoco.community（用”.”表示目录层级关系）顺着目录一步步打开可以看到 CommunityApplication。这是我们执行的基础文件（在此处点击右键运行此文件即可运行整个工程）。src.main.java.cc.smcoco.community路径用于存放所有主要代码。比如创建的各种类，各种功能代码。

  3. src.main.resources（资源目录）： 其中static中存放一些资源文件（css、js等我后来下载的bootstrap文件存放其中），templite文件夹中存放一些模板文件？

  4. pom： community根目录下，pom用于存放依赖。把需要的依赖按固定格式写入后，idea会自动导入、下载需要的资源。

小试牛刀：

使用springtboot做一个简单的网页功能。

功能： 在网页的地址栏写入

[http://自己的网址] + ? + name = [某某值]
# 如：http://community.smcoco.cc?name=dongxia
# 页面显示：Hello,dongxia!

此网页便会把这个某某值变量传递到网页，在页面中展示出来 “Hello，某某值！”

01. 引入需要功能的maven包

在 spring.io/guides （一下称之为教程 ） 网页中搜索thymeleaf（这是我们要用到的功能资源包），在maven位置找到如下格式的代码：

<<dependency>
    <groupId>org.springframework.bootgroupId>
    <artifactId>spring-boot-starter-thymeleafartifactId>
dependency>

放置于工程中pom对应位置，即一堆 

<<dependency>

中，即添加相关依赖。

如图：

此功能实现将所需的资源包加载进pom.xml文件内，idea会自动读取此文件内需要的依赖文件，然后根据弹出的内容选择import，即自动导入好所需要的依赖功能。

此即为maven的功能和用法。

今后使用其他功能（依赖）时，只要此功能在maven库中有存放，即可在 mvnrepository.com 中找到（或者对应功能的网站有提供maven的引用方法），然后找到如上两个dependency标签包裹的内容，把其内容添加至工程的pom.xml文件对应的dependency位置即可。

02:写后端功能

然后我们继续读教程(刚刚的spring里thymeleaf)，找到教程的controller部分，然后在src.main.java.cc.smcoco.community(即刚刚说的放各种代码的目录位置)中右键创建一个名叫controller的package，（如果没有package选项，请检查是否设置的source root根目录） 然后在此包下创建一个IndexController （类的命名首字母要大写，方法则不用）。将对应代码写入。如下：

package cc.smcoco.community.controller;

import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;

@Controller //Controller注解（可以理解为标签），指明此类为Controller类别的类（此处存疑，对“@”的用法不是很明确）。
public class IndexController {
    @GetMapping("/") //获取路径的注解，“/”表示根目录。
	public String greeting(@RequestParam(name="name", required=false, defaultValue="World") String name, Model model) {
		model.addAttribute("name", name);
		return "index"; //将我们在地址栏中 ？name = 某某值的值提取出来，赋值给name变量，然后传给路径下的index网页。
	}

}
\

03:写index网页

继续读教程(刚刚的spring里thymeleaf)，找到其对应greeting.html部分，在src.main.java.resources.templates中创建一个名为index的html文件。（即模板文件），然后写入我们要构建的网页的html代码，即刚刚赋值的部分。（注意，此处我将他对应的greeting修改成了index，因此文件命名也要做改变）

<!DOCTYPE <!DOCTYPE HTML>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title>Getting Started: Serving Web Contenttitle>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
head>
<body>
    <p th:text="'Hello, ' + ${name} + '!'" />
    
body>
html>

04:运行并测试

好了，我们已经把基本的功能代码写完了！现在我们去community里的CommunityApplication右键运行，（或者在idea里右上角的绿色三角运行），然后我们打开浏览器，输入

http://localhost:8080?name=xxx

（注意：因为我们没有对所写的代码进行部署到服务器上，因此我们通过localhost进行本地测试，“？”后面的内容即为我们对网页执行的操作，即把xxx传给name）

然后按回车搜索网页，网页内容显示为 Hello，xxx！

成功！（打教程好累啊！！！TAT不过我相信自己能讲出来，可以促进自己好好地整理和了解！也可能会帮到他人？）

导航栏前端制作

导航栏登录按钮功能实现

0. 思路：

  1. 点击登陆按钮
  2. 社区网页 -> 目标（Github）的callback页面请求authorize
  3. 目标 -> 社区网页 携带code回调
  4. 社区网页 -> 目标（Github）携带code请求access_token
  5. 目标 -> 社区网页user携带access_token

此处我们选择Github提供的API：

  • 点开Github主页，下拉到最底部，找到API点击进入，找到Building a GitHub App or an OAuth App，并点入。

  • 在右侧导航栏中选择Build OAuth Apps并参阅此教程文档。

1. 创建OAuth Apps，获得权限authorize

  1. 重新打开一个Github页面，选择Settings，左侧导航栏选择Developer Settings，在OAuth Apps下选择New OAuth Apps

    输入：

    name： Community

    HomeURL： http://community.smcoco.cc（按最初设定的网址来，详见"创建工程"-> “配置Metadata”）

    callbackURL 地址/callback (此处暂时写本地地址http://localhost:8080/callback)

    点击注册。

2. 获得access token

  1. 绑定“登陆”按钮对应的地址，我们在教程“Authorizing OAuth Apps”中可以看到，点击登陆按钮需要跳转到“ https://github.com/login/oauth/authorize ” 地址。
    在工程中找到templetes.index，在登陆类的地址标签中，输入上述地址，然后输入”?……&……&……”省略号中添加我们地址中要携带的参数，具体如Github提供的教程所述，很多值其实为我们刚刚在Github创建的一个Apps中的参数，找到对应的即可。

此处的意思为，我们要去Github提供的authorize页面，带着我们自己设定的几个参数数值，在其中找到自己的东西并返回。

  1. 在community.controller包下，创建一个AuthorizeController类,内容如下：
package cc.smcoco.community.controller;

import cc.smcoco.community.dto.AccessTokenDTO;
import cc.smcoco.community.dto.GithubUser;
import cc.smcoco.community.provider.GithubProvider;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;

@Controller
public class AuthorizeController {
    //授权控制器
    @Autowired
    private GithubProvider githubProvider;
    @GetMapping("/callback")
    public String Callback(@RequestParam(name="code") String code,
                           @RequestParam(name="state") String state){
        //创建一个AccessTokenDTO对象，并设置其参数，为我们要去Github中找到指定accessToken的标识。
        AccessTokenDTO accessTokenDTO = new AccessTokenDTO();
        accessTokenDTO.setClient_id("cf638083f29d898f388f");
        accessTokenDTO.setClient_secret("20d05cfdb24b92f446de3b5d0d72ea06ce963dd0");
        accessTokenDTO.setCode(code);
        accessTokenDTO.setRedirect_uri("http://localhost:8080/callback");
        accessTokenDTO.setState(state);
        //通过设定好的参数标识，获得accessToken和user信息
        String accessToken = githubProvider.getAccessToken(accessTokenDTO);//GithubProvider类中的方法之一
        GithubUser user = githubProvider.getUser(accessToken);//GithubProvider类中的方法之一
        System.out.println(user.getName());
        return "index";
    }
}

对于中间设置参数的环节，我们在community目录下新建一个与controller包同级的包名叫dto，其中创建一个类名叫“AccessTokenDTO”。

其内容如下：

package cc.smcoco.community.dto;

public class AccessTokenDTO  {
    private String client_id;
    private String client_secret;
    private String code;
    private String redirect_uri;
    private String state;
    //其中下面关于上述参数的get/set方法，我们创建好上方参数以后，使用command + N 快捷键选择get/set直接生成即可。
    public String getClient_id() {
        return client_id;
    }

    public void setClient_id(String client_id) {
        this.client_id = client_id;
    }

    public String getClient_secret() {
        return client_secret;
    }

    public void setClient_secret(String client_secret) {
        this.client_secret = client_secret;
    }

    public String getCode() {
        return code;
    }

    public void setCode(String code) {
        this.code = code;
    }

    public String getRedirect_uri() {
        return redirect_uri;
    }

    public void setRedirect_uri(String redirect_uri) {
        this.redirect_uri = redirect_uri;
    }

    public String getState() {
        return state;
    }

    public void setState(String state) {
        this.state = state;
    }
}

对于AuthorizeController下面两个GithubProvider提供的方法，我们在community目录下新建一个与controller包同级的包名叫provider。内容如下：

package cc.smcoco.community.provider;

import cc.smcoco.community.dto.AccessTokenDTO;
import cc.smcoco.community.dto.GithubUser;
import com.alibaba.fastjson.JSON;
import okhttp3.*;
import org.springframework.stereotype.Component;

import java.io.IOException;

@Component
public class GithubProvider {
    public String getAccessToken(AccessTokenDTO accessTokenDTO){
        MediaType mediaType = MediaType.get("application/json; charset=utf-8");
        OkHttpClient client = new OkHttpClient();
        RequestBody body = RequestBody.create(mediaType, JSON.toJSONString(accessTokenDTO));
        Request request = new Request.Builder()
                .url("https://github.com/login/oauth/access_token")
                .post(body)
                .build();
        try (Response response = client.newCall(request).execute()) {
            String string = response.body().string();
            String token = string.split("&")[0].split("=")[1];
            return token;
        } catch (Exception e) {
            e.printStackTrace();
        }
        return null;
    }
}

此处方法可以参阅OkHttp（相关部分的代码我已直接搬运在上方），直接使用这段代码可能会缺失很多东西，我们用option + enter修复一下。需要的依赖也去maven仓库寻找。

public class GithubUser {
