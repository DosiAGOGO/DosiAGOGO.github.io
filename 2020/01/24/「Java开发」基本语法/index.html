02 类与对象

  • 一个源文件中只能有一个public类（类与文件名对应一致）

  • 一个源文件可以有多个非public类

  • 如果一个类定义在某个包（是什么意思） 中，那么package语句应该在源文件的首行。

  • import语句放在package语句和类定义之间。

  • import语句和package语句对源文件中定义的所有类都有效。在同一源文件中，不能给不同的类不同的包声明。

    父类中声明为 private 的方法，不能够被继承。

04 引用类型

类似于C/C++的指针，指向一个对象，他其实和int之类类似，java的数据类型分为两类：原始数据类型（int、long之类）&引用数据类型。引用数据类型即非标准的那些数据类型。

比如正常我们是

int a = 10

而引用数据类型我们可以有

Site a = new Site("Runoob")。
  • 对象、数组都是引用数据类型。
  • 所有引用类型的默认值都是null。

05 java中的new， XXX a = new XXX（）；

  • 首先解析后半部分，new XXX的作用，XXX是一个类，它是没有实例化的，也就是说，我们定义了一个书的类，类里有标签啊，内容啊，书名啊，但是这是抽象的，只有实例化到某本书，他才有具体对应的书本信息。即标签是四大名著，内容是……，书名是《三国演义》。

new后面跟一个构造方法，来创建一个对象。

  • 再看前半部分 XXX a，即在内存中分配一个变量，名字叫a，这个变量是XXX类型的。
  • 因此，前半部分 XXX a 是创建一个内存空间，等待存放东西；后半部分，new XXX（）是调用XXX（）构造方法将类XXX实例化，创建一个XXX类对应的对象。然后把这个对象赋值给刚刚创建的空间。

使用方法：加括号表达强制转换。

表示将a转换为int类型。

09 非访问修饰符 （什么叫非访问修饰符？）

  • static 修饰符，用来修饰类方法和类变量。静态，就是全局，比如类中的static修饰的变量，就是所有的对象共享的变量。
  • final 修饰符，用来修饰类、方法和变量，final 修饰的类不能够被继承，修饰的方法不能被继承类重新定义，修饰的变量为常量，不可修改。就是常量的意思。
  • abstract 修饰符，用来创建抽象类和抽象方法。抽象方法是一种没有任何实现的方法，该方法的的具体实现由子类提供。子类必须实现父类的所有抽象方法。
  • synchronized（同一时间只能被一个线程访问） 和 volatile（每次被线程访问时，都强制从共享内存中重新读取该成员变量的值） transient（java 虚拟机(JVM)跳过该特定的变量） 修饰符，主要用于线程的编程。

Java小细节补充

@的用法与意义

注解，顾名思义，注解,就是对某一事物进行添加注释说明，会存放一些信息，这些信息可能对以后某个时段来说是很有用处的。

  • @Override:只能用在方法之上的，用来告诉别人这一个方法是改写父类的。

  • @Deprecated:建议别人不要使用旧的API的时候用的,编译的时候会用产生警告信息,可以设定在程序里的所有的元素上.

  • @SuppressWarnings:这一个类型可以来暂时把一些警告信息消息关闭.
    但是，仅仅这三个标注是不能满足我们开发时一些需求的。所以java允许我们自定义注解来使用。

    元注解（给注解用的注解）有：

    1. @Target (注解用在什么地方)
    2. @Retention（在什么级别保存注解）
    3. @Documented（将此注解保存在javadoc中）
    4. @Inherited （允许子类继承父类的注解）
