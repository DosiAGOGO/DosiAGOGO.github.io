「JDBC」知识点

无标签
发布日期:   2020-04-10

架构

B/S架构

Browser/Server 浏览器端、服务器

C/S架构

Client/Server 客户端、服务器

二者区别

C/S
需要更新C，可移植性差

B/S
可以直接获得新内容，可移植性好

JDBC概述

JAVA Database Connetivity通用的SQL数据库存取和操作的公共接口。（API）

实现流程

Java应用程序 -> JDBC接口 -> 具体数据库的JDBC驱动 -> 具体数据库

面向应用的API，面向数据库的API –面向接口编程

程序员面向JDBC接口开发，各数据库厂商面向JDBC接口实现自己数据库的对接

作用

增加移植性，屏蔽了数据库的底层细节。

JDBC程序编写步骤

  1. 导入java.sql包
  2. 附加相应厂商驱动
  3. 加载并注册驱动程序
  4. Connect对象
  5. Statement对象
  6. 执行SQL语句
  7. ResultSet对象（SELECT的结果展示）
  8. 关闭以上对象

JDBC数据库连接

方式一

  1. Driver （驱动-JDBC在不同数据库相对应的驱动，连接哪一种数据库）
  2. url (数据库位置)
  3. Property:user, password (数据库账号密码)
  4. 返回一个connect

方式二

url,user,password写入配置文件 (properties)。加载properties获得相关信息。

【最终方法优点】

  1. 数据与代码分离，实现了解耦
  2. 如果需要修改配置文件信息，避免程序重新打包

使用PreparedStatement实现CRUD操作

CRUD：增删改查

Statement作用

数据库连接(本质Socket连接)向数据库服务器发送命令和SQL语句，并接受其返回的结果。

创建Statement

->使用PreparedStatement

Statement种类：

  1. Statement: 静态SQL语句并返回结果

    缺点：涉及到SQL语句使用程序变量时需要拼串-“sql ‘“+xxx+”‘sql”
    SQL注入，变量和SQL保留字判断错误（or变量被识别成保留字从而导致SQL语句结构改变）。因此使用PreparedStatement替换。

  2. PreparedStatement: SQL语句被预编译并存储在此对象，可供多次高效的执行

    特点：涉及到变量，使用“？”作为占位符，预编译SQL语句，返回PreparedStatement实例。再使用.set…填充。

  1. CallableStatement: SQL存储

PreparedStatement步骤总结

  1. 获取数据库连接（封装Utils）
  2. 预编译sql语句，返回PreparedStatement实例
  3. 填充占位符
  4. 执行
  5. 资源关闭（封装Utils）
//sql语句，使用 ? 作为占位符。
String sql = "";
//sql中放sql语句
PreparedStatement ps = connect.preparedStatement(sql); //connect为上一步产生的数据库连接。
//填充占位符
ps.set..(index,val); 
//执行PreparedStatement
ps.execute(); 
//资源关闭
ps.close(); 
//资源关闭
connect.close();
